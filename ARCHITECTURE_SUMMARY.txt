KEY FINDINGS
================================================================================

1. WHAT THE CODEBASE DOES
   - Analyzes Clash Royale game videos
   - Detects game state (cards, towers, elixir, timer, phase)
   - Runs Decision Transformer (DT) inference to recommend card placements
   - Supports both post-match analysis and real-time processing

2. ARCHITECTURE PATTERN
   - Modular pipeline: Video → Frame → Detection → State → Recommendation
   - Component-based design with clean separation of concerns
   - Pluggable strategy system (MLStrategy vs DTStrategy)

3. CORE TECHNOLOGIES
   - Roboflow (YOLO-based detection for cards/towers)
   - EasyOCR (for reading elixir, timer, multipliers)
   - PyTorch + Transformers (Decision Transformer model)
   - OpenCV (frame processing)
   - NumPy (feature encoding)

================================================================================
CRITICAL COMPONENTS
================================================================================

1. GAME STATE (FrameState)
   Location: /src/video/video_analyzer.py
   Tracks: timestamp, elixir, timer, game_phase, towers, detections, cards
   Updated: Per frame via VideoAnalyzer or LivePipeline

2. DETECTION PIPELINE
   Location: /detection_test.py
   Model: Roboflow torchroyale/4 (YOLO)
   Output: Detections with pixel & tile coordinates
   Maps: Pixel coords → 18x32 tile grid via CoordinateMapper

3. GAME PHASE TRACKER
   Location: /src/game_state/game_phase.py
   States: SINGLE → DOUBLE → (SUDDEN_DEATH) → TRIPLE → GAME_OVER
   Input: multiplier icon (x1/x2/x3), timer, tower status
   Effect: Determines elixir regen rate

4. OPPONENT ELIXIR TRACKER
   Location: /src/recommendation/elixir_manager.py
   Calculation: 5 (start) + time*regen_rate - card_costs
   Regen rates vary by phase (2.8s, 1.4s, 0.933s per elixir)

5. FEATURE ENCODER
   Location: /src/data/feature_encoder.py
   Input: FrameState dict
   Output: 97-dim fixed-size vector
   Components: elixir(2) + phase(4) + time(1) + towers(6) + detections(80) + hand(4)

6. DT STRATEGY
   Location: /src/recommendation/strategy.py
   Model: Decision Transformer checkpoint (.pt file)
   Context: Maintains rolling window of past states/actions/RTG
   Input: Feature vector + target return
   Output: (card_index, tile_position)

================================================================================
DATA FLOW
================================================================================

Video Analysis Pipeline:
  Video File
    ↓ [VideoProcessor]
  Frame (cv2.Mat)
    ├─ [DetectionPipeline] → detections
    ├─ [DigitDetector] → elixir, timer, multiplier
    ├─ [GamePhaseTracker] → game_phase
    ├─ [OpponentElixirTracker] → opponent_elixir
    ├─ [TowerHealthDetector] → tower_health
    ↓
  FrameState (complete game state)
    ↓ [feature_encoder.encode()]
  Feature Vector (97 dims)
    ↓ [DTStrategy.recommend()]
  Output: (card_name, tile_x, tile_y)

Real-Time Pipeline (LivePipeline):
  Live Frame Stream
    ├─ [OCR cached every 3 frames]
    ├─ [Detection cached every 2 frames]
    ├─ [Tower health cached every 15 frames]
    ↓ [Per frame]
  game_state_dict
    ├─ elixir (int)
    ├─ opponent_elixir (float)
    ├─ timer (seconds)
    ├─ game_phase (enum)
    ├─ tower_health (dict)
    └─ processing_time (float)

================================================================================
PERFORMANCE CHARACTERISTICS
================================================================================

LivePipeline (Real-time):
  Target FPS: 30
  Actual FPS: 15-30
  Caching:
    - OCR: every 3 frames (10 Hz)
    - Detection: every 2 frames (15 Hz)
    - Tower health: every 15 frames (2 Hz)
  Processing time per frame: ~30-50ms

VideoAnalyzer (Batch):
  Frame skip: configurable (default 6 = ~5 FPS from 30 FPS)
  Processing: ~1-5 FPS depending on frame skip
  Output: JSON analysis file + optional annotated frames
